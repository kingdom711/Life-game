# 전체 프로젝트 디자인 리뉴얼 계획

## 분석된 잠재적 문제점

### 높은 리스크

| 문제 | 영향 범위 | 해결 방안 ||------|----------|----------|| `body` 배경 이미지 제거 | 전체 페이지 | 새 그라데이션 배경 시스템으로 대체 || CSS Variables 변경 | 변수 참조하는 모든 컴포넌트 | 단계별 변경 및 테스트 || 하드코딩된 색상값 (약 200개 이상) | Tailwind 클래스 및 인라인 스타일 | 검색/치환으로 일괄 수정 |

### 중간 리스크

| 문제 | 영향 범위 | 해결 방안 ||------|----------|----------|| 모달/오버레이 자체 배경색 | CalibrationModal, HazardQuestModal, MonthlyAttendanceModal 등 | 공통 모달 스타일 정의 || Tailwind + 커스텀 CSS 충돌 | 스타일 우선순위 | CSS 구조 정리, 중복 제거 |

### 기능 영향 없음 확인

- 라우팅, 상태 관리, API 호출 등 기능적 코드는 스타일과 분리되어 있어 영향 없음
- 이벤트 핸들러, 데이터 로직은 JSX 구조만 유지하면 정상 동작

## 구현 계획

### Phase 1: 디자인 시스템 기반 구축

**1.1 CSS Variables 재정의** - [`src/styles/index.css`](src/styles/index.css)현재 CSS Variables를 유지하면서 새로운 색상 체계 추가:

- 새 배경 색상: 순수 그라데이션 (Sky Blue → White → Lavender)
- 페이지별 테마 변수 추가 (Dashboard: 블루톤, Shop: 인디고톤 등)

**1.2 배경 이미지 제거 및 대체** - [`src/styles/index.css`](src/styles/index.css)

```css
body {
  background: linear-gradient(135deg, #f0f9ff 0%, #ffffff 50%, #f5f3ff 100%);
  /* bg_bright_construction.png 제거 */
}
```

**1.3 App.jsx 전역 배경 효과 업그레이드** - [`src/App.jsx`](src/App.jsx)현재 floating orb 효과를 더 세련되게 개선---

### Phase 2: 공통 컴포넌트 스타일 통일

**2.1 카드 컴포넌트 표준화**

- 공통 `.card` 클래스 개선 (Glassmorphism + 부드러운 그림자)
- 호버 효과 통일

**2.2 버튼 스타일 통일**

- `.btn-*` 클래스 그라데이션 및 그림자 효과 개선
- 네온 하이라이트 옵션 추가

**2.3 모달 스타일 통일**

- 공통 모달 오버레이 스타일
- CalibrationModal의 High-Tech 요소를 선택적으로 적용

---

### Phase 3: 페이지별 테마 적용

각 페이지의 특성은 유지하되 통일된 디자인 언어 적용:| 페이지 | 테마 컬러 | 특성 유지 ||--------|----------|----------|| Dashboard | Blue/Indigo | 통계 카드 그라데이션 || Shop | Indigo/Purple | 희귀도별 글로우 효과 || Inventory | Cyan/Purple | High-Tech Lab 컨셉 (CalibrationModal 연계) || Profile | Green/Teal | 성장 시각화 || Quests | Orange/Amber | 게임적 느낌 |---

### Phase 4: 세부 조정 및 최적화

- 반응형 디자인 확인 (모바일/태블릿)
- 애니메이션 성능 최적화
- 브라우저 호환성 테스트
